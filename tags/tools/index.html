配置 Emacs

为什么诸位都如此热衷于 vscode 啊？我随便一翻都能看到 inf 篇介绍 vscode 配置的文章。 不如来玩点神秘的：Emacs。 什么是 Emacs？ Emacs 是一款由 GNU 开源软件基金会完全自主研发的开源文本编辑器。在这款编辑器中，你将在一个叫做“buffer”的环境中冒险，与 elisp 邂逅，并探明“Emacs pinky”的真相…… 不开玩笑了。Emacs 是知名老牌文本编辑器，与 vi/vim 并列，长期处于鄙视链的顶端。主要优势是无与伦比的可定制性——它是 lisp machine 思想的继承者，整个编辑器内核和其他各种功能都使用 elisp 编写，并且有内嵌的 elisp 解释器。用户可以自由地修改这些代码，或者增加属于自己的函数，从而定制编辑器的行为。它的可定制性是图灵完备的：理论上，你可以用它来做任何事情，包括但不限于浏览网页，管理日程，与朋友交流，或是原神启动。 如何获取？ 首先，你需要有一个 Emacs。最简单的方法是访问 GNU Emacs 项目官网或者 github 仓库然后下载。你也可以用你的 msys2 pacman 或者 winget 等等包管理器下载它。具体的包的名字我就不提供了，反正包管理器随便搜搜就能搜到。 然后，你就可以使用它了。不过这时的 Emacs 没有任何配置，外观很丑陋，功能也很匮乏。 我们需要对它进行一些配置来获得想要的效果。 如何配置？ 好问题，你肯定听说过 Emacs 的配置非常困难，不过事实上作为一个新手（就像我）要快速上手还是很简单的。 要义在于不要纯靠自己配置。靠自己攒配置消耗的时间精力太大了，在你配出好用的编辑器前就会失去兴趣。 我们可以搜寻一些别人做好的 Emacs 发行版。比如经典的 doom Emacs、spacemacs、centaur Emacs 或者 Prelude 等等。其中 doom Emacs 和 spacemacs 默认开启 evil-mode（vim 模式），使用体验类似 vim。我没用过 vim 不会用，所以自然放弃了它们。centaur Emacs 和 Prelude 则坚持原生体验，Prelude 比较轻量级，并且为用户保留了自己进一步配置的能力，所以我选择了它。 ...
