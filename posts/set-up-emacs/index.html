配置 Emacs

October 27, 2025

为什么诸位都如此热衷于 vscode 啊？我随便一翻都能看到 inf 篇介绍 vscode 配置的文章。

不如来玩点神秘的：Emacs。

什么是 Emacs？#

Emacs 是一款由 GNU 开源软件基金会完全自主研发的开源文本编辑器。在这款编辑器中，你将在一个叫做“buffer”的环境中冒险，与 elisp 邂逅，并探明“Emacs pinky”的真相……

不开玩笑了。Emacs 是知名老牌文本编辑器，与 vi/vim 并列，长期处于鄙视链的顶端。主要优势是无与伦比的可定制性——它是 lisp machine 思想的继承者，整个编辑器内核和其他各种功能都使用 elisp 编写，并且有内嵌的 elisp 解释器。用户可以自由地修改这些代码，或者增加属于自己的函数，从而定制编辑器的行为。它的可定制性是图灵完备的：理论上，你可以用它来做任何事情，包括但不限于浏览网页，管理日程，与朋友交流，或是原神启动。

如何获取？#

首先，你需要有一个 Emacs。最简单的方法是访问 GNU Emacs 项目官网或者 github 仓库然后下载。你也可以用你的 msys2 pacman 或者 winget 等等包管理器下载它。具体的包的名字我就不提供了，反正包管理器随便搜搜就能搜到。

然后，你就可以使用它了。不过这时的 Emacs 没有任何配置，外观很丑陋，功能也很匮乏。

我们需要对它进行一些配置来获得想要的效果。

如何配置？#

好问题，你肯定听说过 Emacs 的配置非常困难，不过事实上作为一个新手（就像我）要快速上手还是很简单的。

要义在于不要纯靠自己配置。靠自己攒配置消耗的时间精力太大了，在你配出好用的编辑器前就会失去兴趣。

我们可以搜寻一些别人做好的 Emacs 发行版。比如经典的 doom Emacs、spacemacs、centaur Emacs 或者 Prelude 等等。其中 doom Emacs 和 spacemacs 默认开启 evil-mode（vim 模式），使用体验类似 vim。我没用过 vim 不会用，所以自然放弃了它们。centaur Emacs 和 Prelude 则坚持原生体验，Prelude 比较轻量级，并且为用户保留了自己进一步配置的能力，所以我选择了它。

对于 Prelude，你需要将官方项目下载下来，放进你 Emacs 配置文件的目录中（Windows 中默认是 C:\Users\<name>\AppData\Roaming\.emacs.d\，在 powershell 中使用 $env:appdata 找到你放 APP 配置文件的目录，然后在这个目录中新建 /.emacs.d/ 子目录，放置你的配置文件）。这个过程可以使用 git clone <project address> ~/.emacs.d/ 命令自动化，省去操作 GUI 的功夫。

这个目录中的 init.el 是我们常说的“Emacs 配置文件”，不过此时 Prelude 占用了它来完成 Prelude 的配置。我们最好不要改动它，如果要自己进一步配置，Prelude 为我们预留了 ~/.emacs.d/personal/ 目录来放置自己的配置文件，其中 custom.el 是你的个人主配置。

哦，你问 ~/.emacs.d/ 中的波浪线是什么？它是一个占位符，代表我们的配置文件所处的位置。比如 Windows 下就是 $env:appdata 环境变量的值。

然后启动 Emacs（使用 runemacs.exe 体验会比较好，没有那个烦人的黑色控制台）。Emacs 会自动运行 init.el（也就是它的配置文件），Prelude 提供的配置文件会指引它去下载和编译一系列插件（通过它的 elpa 包管理器）并初始化它们的状态。过程中 Emacs 可能会卡住，问题不大，等待即可。

之后你的 Emacs 就配置好了。尽情玩耍吧。

如何使用？#

就把它当做一个普通的文本编辑器用就行了。和 vscode 或者 sublime 没有区别。

不过还是有些奇特的地方的，比如说快捷键，比如说外观，等等。但这些都不是问题，去问你的 genAI Emacs 最基本最常用的快捷键，然后你花几分钟就学会了。都 2025 年了，这样的小事不应该成为门槛。
而外观……Prelude 默认的主题我就很喜欢，和 CLion 体验有点类似，至少比 dev-C++ 好看的多。

Prelude 装的插件有很多强大的功能，在使用中你自然会逐渐接触和使用。

然后就是我的配置，其实就是 Prelude。但是改了 $O(1)$ 个小地方：设置了字符集为 Unicode，编码为 UTF-8，设置了中文字体（不然中文完全没法看），关掉了 whitespace 对长行的提示（一种很难看的紫色），以及因为我用 Emacs 来写 scheme 和 racket，启用了 scheme 和 racket 对应的模式，设置了 geiser 使用 chez scheme 作为 scheme 实现，修复了 scheme mode 语法高亮的失效问题。

我的配置文件在 https://github.com/litjohn/my-emacs-settings 中，可以自取。注意我由于个人喜好使用了 Jetbrains Mono 作为默认字体，如果你没有安装的话可以改成 Consolas，或者也可以改成你自己喜欢的字体。

你说我没有配置 C++ 的 major mode 和相关命令？不是哥们，写 C++ 这样的主流语言为什么不用 JetBrains 产品？CLion 已经免费了！我用 Emacs 只是因为它写 scheme 和 racket 顺手而已。
